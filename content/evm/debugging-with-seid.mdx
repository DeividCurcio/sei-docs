# Debugging with seid

## Transaction Template Generation

The `--generate-only` flag transforms any Sei CLI transaction command into a template generator, creating complete transaction structures without broadcasting them. Through the `--generate-only` flag and Foundry's `cast` tool, developers can craft, analyze, and debug transactions across in the EVM environment.

### Basic Command Pattern

The general pattern follows this structure:

```bash copy
seid tx <module> <action> <parameters> --from <key> --generate-only
```

### Generating EVM Transaction Templates

To generate an EVM transaction template, use the `evm` module with `--generate-only`. Here's an example registering an EVM pointer:

```bash copy
seid tx evm register-evm-pointer NATIVE \
factory/sei1ep3f207kt7julz9tjwxp2x8kluj0y9l6u0fume/gptw \
--gas-fee-cap=100000000000 \
--gas-limit=2500000 \
--evm-rpc=https://localhost:8545 \
--from tf \
--generate-only
```

<Callout type="info" emoji={<IconInfoCircle className="h-5 w-5 mt-1" />}>
  This command returns a transaction hash that you can analyze further using Foundry's `cast` tool.
</Callout>

## Analyzing EVM Transactions with Cast

Once you have a transaction hash, you can use Foundry's `cast` command to inspect the transaction details:

```bash copy
cast tx 0x5010e6600e67f04a9bc3d3b670a7c2de380b180713d9a014a5dbd76b7e2190f1 \
--rpc-url=https://localhost:8545
```

## Template Generation for Different Transaction Types

Generate templates for various transaction types:

```bash copy
# EVM contract interaction
seid tx evm send-tx \
--evm-rpc=http://localhost:8545 \
--gas-limit=2500000 \
--from=mykey \
--generate-only > evm_template.json
```
