# Sei Giga Overview

## Introduction

Sei Giga represents an ambitious evolution in blockchain technology, designed to address the persistent challenges of the "blockchain trilemma": the difficulty of simultaneously optimizing for security, decentralization, and performance. Through a combination of implemented optimizations and planned architectural innovations, Sei aims to achieve unprecedented performance while maintaining the security guarantees and developer experience of Ethereum.

The platform's vision centers on achieving up to 5 gigagas throughput (5 billion gas per second) with sub-second finality, enabling web2-scale applications on a verifiable public ledger. This document provides a comprehensive technical overview of both the current implementation and the roadmap toward this goal.

<Callout type="info">
**Implementation Status**: Sei currently operates as a high-performance blockchain with Twin Turbo consensus optimizations, parallel execution capabilities, and SeiDB storage. The full Sei Giga architecture, including Autobahn consensus and 5 gigagas throughput, represents the platform's technical roadmap and vision.
</Callout>

## Current Architecture

Sei's current implementation delivers significant performance improvements through three core components:

### 1. Twin Turbo Consensus
The current consensus mechanism achieves ~400ms block times through aggressive optimization of the Tendermint BFT consensus engine. Key features include:
- Pipelined block processing
- Optimistic transaction execution during consensus
- Aggressive timeout configurations
- Parallel transaction decoding and validation

[Learn more about Twin Turbo Consensus →](/learn/twin-turbo-consensus)

### 2. Parallel Execution Engine
Sei implements Optimistic Concurrency Control (OCC) for parallel transaction execution:
- Multiple CPU cores process non-conflicting transactions simultaneously
- Automatic conflict detection and resolution
- Maintains deterministic execution order
- Compatible with standard EVM semantics

[Learn more about the Parallelization Engine →](/learn/parallelization-engine)

### 3. SeiDB Storage System
A specialized database optimized for blockchain workloads:
- Multi-level caching for hot state data
- Optimized Merkle Patricia Trie implementation
- Concurrent state access controls
- Efficient state versioning and pruning

[Learn more about SeiDB →](/learn/seidb)

## Sei Giga Vision: Architectural Foundations

The complete Sei Giga architecture, as outlined in the technical whitepaper, rests on three key innovations that together enable its target performance characteristics:

### 1. Asynchronous Execution Model

A fundamental design principle of Sei Giga is the complete separation of consensus and execution. Unlike traditional blockchains where these processes are tightly coupled, Sei Giga's vision is to reach consensus solely on the ordering of transactions within a block, not the resulting state change.

This approach leverages the key property of deterministic execution:

**Lemma 1 (Deterministic Execution)**: Given the same initial state (S<sub>init</sub>) and the same ordered sequence of transactions (tx<sub>1</sub>, tx<sub>2</sub>, ..., tx<sub>n</sub>), all honest nodes executing these transactions will arrive at the exact same final state (S<sub>final</sub>).

Key benefits of this model:
- **Non-blocking consensus**: Heavy computational blocks don't delay consensus
- **Pipelined processing**: Multiple blocks can be in different stages simultaneously
- **Optimized resource usage**: Consensus and execution use different hardware resources

The protocol operates through two complementary layers:

**Data Dissemination Layer (Lanes)**:
- Each validator maintains an independent lane for proposing transaction batches
- Three-step certification process: Propose → Vote → Proof of Availability (PoA)
- Chained proposals ensure availability of historical data

**Consensus Layer (Cut of Tips)**:
- Periodic ordering of latest certified tips from all lanes
- Two-phase BFT agreement with fast and slow paths
- Pipelined slots for minimal latency

## Developer Experience

Sei maintains full EVM compatibility while providing enhanced capabilities:

### Standard EVM Features
- Complete support for Solidity and Vyper
- Standard JSON-RPC endpoints
- Compatible with existing tools (Hardhat, Foundry, Remix)
- Familiar development workflow

### Enhanced Capabilities
- Sub-second transaction finality
- Predictable gas costs
- High throughput for complex applications
- Advanced storage access patterns

### Migration Path

Existing Ethereum applications can migrate to Sei with minimal changes:

```solidity
// Standard ERC-20 works without modification
contract MyToken is ERC20 {
    constructor() ERC20("MyToken", "MTK") {
        _mint(msg.sender, 1000000 * 10**decimals());
    }
}

// Leverage fast finality for better UX
contract FastDeFi {
    // Users experience near-instant confirmation
    function swap(address tokenIn, address tokenOut, uint256 amount) external {
        // Swap executes and confirms in <1 second
        // No need for complex pending state management
    }
}
```
