# Node Operator Guide

## Building from Source

### Install Go

```bash
wget https://go.dev/dl/go1.21.1.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.21.1.linux-amd64.tar.gz

echo 'export GOROOT=/usr/local/go' >> ~/.profile
echo 'export GOPATH=$HOME/go' >> ~/.profile
echo 'export GO111MODULE=on' >> ~/.profile
echo 'export PATH=$PATH:/usr/local/go/bin:$HOME/go/bin' >> ~/.profile
source ~/.profile
```

### Build Sei

```bash
git clone https://github.com/sei-protocol/sei-chain.git
cd sei-chain
git checkout [version]  # See version table below
make install
```

For available versions, see our [version table](../references/versions).

## Network Setup

### Initialize Node

```bash
# Set network variables
CHAIN_ID="pacific-1"  # or "atlantic-2" for testnet
MONIKER="your-node-name"

# Initialize
seid init $MONIKER --chain-id $CHAIN_ID
```

### Configure Node

```bash
# Get genesis
wget -O $HOME/.sei/config/genesis.json https://raw.githubusercontent.com/sei-protocol/testnet/main/$CHAIN_ID/genesis.json

# Set seeds/peers
PEERS="id1@peer1:26656,id2@peer2:26656"
sed -i.bak -e "s/^persistent_peers *=.*/persistent_peers = \"$PEERS\"/" $HOME/.sei/config/config.toml
```

See [Network Configuration](./node-configuration#network) for peer selection strategies.

## Database Setup

### Enable Sei-DB

```toml
# ~/.sei/config/app.toml
[state-commit]
sc-enable = true
sc-async-commit-buffer = 100

[state-store]
ss-enable = true
ss-backend = "pebbledb"
```

## Service Configuration

### System Limits

```bash
# /etc/security/limits.conf
$USER             soft    nofile          65535
$USER             hard    nofile          65535
```

### Service File

```ini
# /etc/systemd/system/seid.service
[Unit]
Description=Sei Node
After=network.target

[Service]
User=$USER
ExecStart=$(which seid) start
Restart=always
RestartSec=3
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target
```

### Start Service

```bash
sudo systemctl daemon-reload
sudo systemctl enable seid
sudo systemctl start seid
```

## Monitoring Setup

### Basic Checks

```bash
# Sync status
seid status | jq .SyncInfo

# Node info
seid status | jq .NodeInfo

# Connection info
seid status | jq .ValidatorInfo
```

For advanced monitoring, see our [Best Practices Guide](./best-practices#monitoring).

## Key Management

### Create Keys

```bash
# Create new key
seid keys add <name>

# Restore existing key
seid keys add <name> --recover
```

See our [Security Guide](./best-practices#key-security) for key management best practices.

## Common Operations

### Update Binary

```bash
cd sei-chain
git fetch --all
git checkout [new-version]
make install
sudo systemctl restart seid
```

### Reset Node

```bash
sudo systemctl stop seid
seid tendermint unsafe-reset-all --keep-addr-book
sudo systemctl start seid
```

For more maintenance tasks, see our [Operations Guide](./best-practices#maintenance).
